# Pokemon CLI - 设计文档

## 1. 项目概述

### 项目名称
**Pokemon CLI** - 使用 Ink 开发的命令行宝可梦游戏

### 核心功能
- 回合制战斗系统
- 宝可梦捕捉与养成
- 地图探索（城市 + 道路 + 迷宫）
- 完整游戏循环：图鉴、队伍、商店、存档

### 目标用户
- 宝可梦爱好者
- CLI 游戏玩家
- 喜欢复古体验的用户

---

## 2. 技术架构

### Monorepo 结构
```
pokemon/
├── packages/
│   ├── cli/           # Ink CLI 游戏主程序
│   ├── editor/        # Tauri 整合编辑器
│   │   ├── sprite/    # 精灵图案编辑器
│   │   ├── map/       # 地图编辑器（城市+道路）
│   │   ├── pokedex/   # 图鉴编辑器
│   │   ├── move/      # 招式编辑器
│   │   └── item/      # 物品编辑器
│   ├── core/          # 核心逻辑、数据类型、游戏引擎
│   ├── i18n/          # 国际化（中→英→日）
│   └── maze/          # 噪声图迷宫生成算法
├── pnpm-workspace.yaml
└── package.json
```

### 核心技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| CLI 游戏 | Ink + React | 命令行 UI 框架 |
| 编辑器 | Tauri + React | 桌面应用框架 |
| 数据获取 | PokéAPI | 宝可梦数据源 |
| 图片转 ASCII | image-to-ascii | PNG → ASCII 转换 |
| 精灵显示 | ink-picture | 终端图片渲染 |
| 迷宫生成 | simplex-noise | 噪声图算法 |
| 包管理 | pnpm | Monorepo 管理 |

## 3. 功能模块

### 3.1 CLI 游戏 (cli)

#### 3.1.1 战斗系统
- 属性克制（18 种属性）
- 技能系统（PP、威力、命中率、属性）
- 经验值与升级
- 捕捉机制（球种选择）

#### 3.1.2 探索系统
- 世界地图（城市连接）
- 迷宫探索（噪声图生成）
- 随机遇敌
- 训练师对战

#### 3.1.3 养成系统
- 队伍管理（最多 6 只）
- 背包系统（道具、精灵球）
- 图鉴系统（捕捉记录）

#### 3.1.4 存档系统
- **自动存档**：每场战斗后自动保存
- **手动存档**：玩家可使用 `save` 命令
- **存档位置**：`~/.pokemon-cli/save.json`
- **存档版本**：包含版本号用于迁移
- **备份机制**：保留最近 3 个存档

#### 3.1.5 队伍系统
- 队伍管理（最多 6 只）
- PC 盒子（最多 100 只，扩展用）
- 满队伍时：提示放生/传送至 PC 或取消捕捉

### 3.2 整合编辑器 (editor)

#### 3.2.1 精灵编辑器 (Sprite Editor)
- 画布绘制（像素/字符）
- **从 PokéAPI 导入精灵 → 程序化转 ASCII**
- **ASCII 艺术是核心视觉风格，不仅仅是回退方案**
  - 彩色字符画（ANSI 颜色）
  - 半块渲染（▄▀）增强细节
  - 独特复古像素美学
- 手动编辑 ASCII 图案
- 预览效果

#### 3.2.2 地图编辑器 (Map Editor)
- 城市布局编辑（瓦片系统）
- 城市间道路连接
- 区域类型标记（草地、水域、山脉）
- 建筑标记（商店、训练馆）

#### 3.2.3 图鉴编辑器 (Pokedex Editor)
- 宝可梦基础数据（名称、属性、HP、攻防速）
- 进化链编辑
- 捕捉地点设置

#### 3.2.4 招式编辑器 (Move Editor)
- 招式属性
- 威力、PP、命中率
- 效果描述

#### 3.2.5 物品编辑器 (Item Editor)
- 道具名称与描述
- 效果参数

### 3.3 核心库 (core)

- 游戏状态管理
- 战斗逻辑
- 数据验证
- 类型定义

### 3.4 国际化 (i18n)

**优先级**：中文 → English → 日本語

- 按此顺序实现界面文本
- 宝可梦名称、招式名称、物品名称支持多语言
- 由中文开发者社区贡献翻译

### 3.5 经济系统
- **金钱获取**：战斗胜利、道具出售
- **商店系统**：道具购买/出售
- **定价策略**：基于道具稀有度

### 3.6 训练师 AI
- 野生宝可梦：随机技能
- 训练师对战：使用策略（技能选择、道具使用）

### 3.7 迷宫生成 (maze)

- Simplex Noise 算法
- 房间生成（5-10 个房间）
- 出口/入口放置
- 宝藏/敌人分布

### 3.8 游戏结束条件
- 训练师全灭：送至最近医院复活，扣除金钱
- 通关：挑战冠军，完成游戏

## 4. 数据流

### 4.1 精灵流程（核心方案）
```
PokéAPI PNG → image-to-ascii 转换 → 存储 ASCII → Ink 直接渲染
```
- 程序化转换作为初始素材
- 玩家可手动编辑优化
- Ink Text 组件直接渲染字符画

### 4.2 精灵流程（可选方案）
```
PokéAPI PNG → ink-picture → iTerm2/Kitty 终端渲染
```

### 4.2 游戏存档
```
内存 → JSON → ~/.pokemon-cli/save.json
```

### 4.3 编辑器 → 游戏
```
1. 编辑器导出 JSON 到 packages/core/data/
2. CLI 启动时读取最新数据文件
3. 可选：添加 --reload-data 强制刷新缓存
```

### 4.4 异常处理
- **网络异常**：PokéAPI 请求超时使用本地缓存数据
- **完全离线**：切换至纯离线模式（仅使用预置数据）
- **存档损坏**：提示恢复备份或重新开始
- **数据文件缺失**：自动生成默认数据

## 5. 精灵显示策略

### 核心方案：ASCII 艺术（主要显示方式）
- **彩色字符画**：ANSI 颜色渲染
- **半块渲染**：▄▀ 字符增强细节
- 独特复古像素美学
- 每个宝可梦都有专属 ASCII 图案

### 高级方案：ink-picture（可选增强）
- 支持 iTerm2/Kitty 协议终端显示真彩色图片
- Windows Terminal 支持 iTerm2 协议
- 可选开启，非必须

### 通用回退
- 纯 ASCII 字符（无颜色）
- Emoji 备选（⚡🔥💧🌿）
